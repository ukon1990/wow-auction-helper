Hi, I m sorry to announce it, but I have decided not to maintain or host this project anymore. It's wistful, but I just do not feel like I have the time and energy to maintain the project. So as of {{endOfService | date: 'medium'}}, the project will be shut down.

A contributing factor is also the lack of documentation and data provided by the Blizzard API. Especially in regards to Recipes. Also in periods it’s a lot of work in addition to my daytime job + family and all that. For example, there is no information about what items belongs to the new recipes (that I've found), so I have to manually connect them. And that makes me insecure about the validity of the data. And makes it harder to make the project live by itself. In addition to that, there is no API for the classic recipes. No official documentation of item level curves or pricing curves.

If anyone wishes to take up the mantle, feel free to fork the code base, and base something off it. If you manage to make it better, great! But if you do, I'd appreciate it if you do not use ads(especially not the sketchy ones), and keep it open source.

I'd like to thank everyone who has used the app.

Hosting and development

The app is hosted on AWS, with S3, Lambda, DynamoDB, RDS (not in AWS). The code base does contain code from when I was a student, but I don’t think that should be an issue.

You can find the the code and documentation here.

If you’ve got some AWS knowledge you should be able to get it up and running. Everything you need should be documented, but would require AWS knowledge. But it should also be possible to convert it to.a regular nodes app with cron jobs.
